# FinSight 测试报告总汇 Spec

## Why
需要对六个模块的测试报告进行汇总，生成一份完整的测试报告文档，并整理所有发现的问题和错误，方便开发团队进行后续修复工作。

## What Changes
- 汇总六个模块的测试结果（认证模块、系统支撑模块、算法引擎模块、配置管理模块×2、可视化报表模块×2）
- 生成完整的测试数据统计
- 整理所有发现的问题和错误，生成错误列表
- 按模块和严重程度分类问题
- 提供修复建议和优先级

## Impact
- 生成文档：docs/test report/测试报告总汇.md
- 影响范围：所有测试模块的错误追踪

## ADDED Requirements

### Requirement: 总测试报告生成
系统应生成一份完整的测试报告总汇，包含以下内容：

#### Scenario: 汇总测试统计
- **WHEN** 需要汇总所有模块测试结果
- **THEN** 报告应包含总测试用例数、总通过数、总失败数、总体通过率

#### Scenario: 错误列表汇总
- **WHEN** 汇总所有发现的问题
- **THEN** 报告应按模块分类列出所有错误，包含错误ID、描述、严重程度、影响接口、建议修复方案

3. 算法引擎模块: 投资方案保存时中文乱码

8. 配置管理模块: 创建记录时返回id为null

4. 配置管理模块: 知识库导入功能未实现
