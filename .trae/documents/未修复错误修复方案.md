## 1. 错误概述

|--------|----------|----------|----------|

## 2. 修复方案

### 2.1 ERR-007 & ERR-008：预警记录表缺失

**问题分析**：
- `WarningRecordMapper.java`中的SQL查询使用的是`algo_warning_record`表
- 数据库中该表不存在，导致500错误

**修复步骤**：

1. 创建数据库初始化SQL脚本 `V1__create_algo_warning_record_table.sql`
2. 在数据库设计文档中找到表结构定义：
   - 字段：id, indicator, indicator_code, level, current_value, yellow_threshold, orange_threshold, red_threshold, trend, suggestion, status, period, triggered_at
3. 确保Mapper的SQL与表结构匹配

**需要创建的文件**：
```
src/main/resources/db/migration/V1__create_algo_warning_record_table.sql
```

**SQL内容**：
```sql
CREATE TABLE IF NOT EXISTS algo_warning_record (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    indicator VARCHAR(100) NOT NULL COMMENT '指标名称',
    indicator_code VARCHAR(50) COMMENT '指标编码',
    level VARCHAR(20) COMMENT '预警级别',
    current_value DECIMAL(10,2) COMMENT '当前值',
    yellow_threshold VARCHAR(50) COMMENT '黄色阈值',
    orange_threshold VARCHAR(50) COMMENT '橙色阈值',
    red_threshold VARCHAR(50) COMMENT '红色阈值',
    trend VARCHAR(20) COMMENT '趋势',
    suggestion TEXT COMMENT '建议',
    status VARCHAR(20) DEFAULT 'active' COMMENT '状态',
    period VARCHAR(50) COMMENT '期间',
    triggered_at DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '触发时间'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='预警记录表';
```

### 2.2 ERR-012 & ERR-013：PDF报告表缺失

**问题分析**：
- `PdfReportMapper.java`中的SQL查询使用的是`report_pdf_report`表
- Mapper在insert方法上已有`@Options(useGeneratedKeys = true, keyProperty = "id")`注解
- 数据库中该表不存在，导致500错误

**修复步骤**：

1. 创建数据库初始化SQL脚本 `V2__create_report_pdf_report_table.sql`
2. 在数据库设计文档中找到表结构定义
3. 确保Mapper的SQL与表结构匹配

**需要创建的文件**：
```
src/main/resources/db/migration/V2__create_report_pdf_report_table.sql
```

**SQL内容**（参考已有SQL文件 `docs/test report/中断情况评估/create_pdf_report_table.sql`）：
```sql
CREATE TABLE IF NOT EXISTS report_pdf_report (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(255) NOT NULL COMMENT '报告名称',
    type VARCHAR(50) NOT NULL COMMENT '报告类型',
    period VARCHAR(50) COMMENT '报告期间',
    status VARCHAR(50) DEFAULT 'pending' COMMENT '状态',
    file_size VARCHAR(50) COMMENT '文件大小',
    creator_name VARCHAR(100) COMMENT '创建者',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='PDF报告记录表';
```

### 2.3 ERR-016：投资方案保存时中文乱码

**问题分析**：
- `InvestmentSchemeMapper.java`的insert方法使用注解方式
- 需要确保数据库连接使用utf8mb4字符集
- 需要确认数据库表`algo_investment_scheme`存在且字符集正确

**修复步骤**：

1. 确认数据库连接配置中使用了utf8mb4字符集
2. 创建数据库初始化SQL脚本 `V3__create_algo_investment_scheme_table.sql`
3. 确保表使用utf8mb4字符集

**需要创建的文件**：
```
src/main/resources/db/migration/V3__create_algo_investment_scheme_table.sql
```

**SQL内容**（参考数据库设计文档）：
```sql
CREATE TABLE IF NOT EXISTS algo_investment_scheme (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    project_name VARCHAR(128) NOT NULL COMMENT '项目名称',
    initial_investment DECIMAL(18,2) NOT NULL COMMENT '初始投资额',
    discount_rate DECIMAL(10,4) NOT NULL COMMENT '折现率(%)',
    cash_flows JSON COMMENT '各年净现金流',
    npv DECIMAL(18,2) COMMENT '净现值',
    irr DECIMAL(10,4) COMMENT '内部收益率(%)',
    static_payback DECIMAL(10,2) COMMENT '静态回收期(年)',
    dynamic_payback DECIMAL(10,2) COMMENT '动态回收期(年)',
    created_by VARCHAR(64) COMMENT '创建人',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='投资评估方案表';
```

**额外检查**：
- 检查`application.yaml`中数据库URL是否包含字符集参数：`characterEncoding=utf8mb4`

### 2.4 ERR-022：创建记录时返回id为null

**问题分析**：
- 根据错误修复报告，这是数据库表自增ID配置问题
- Mapper中insert方法已添加@Options注解
- 需要确认数据库表的主键自增配置是否正确

**修复步骤**：

1. 检查以下Mapper的insert方法是否都有@Options注解：
   - `FormulaMapper.java`
   - `ThresholdMapper.java`
   - `KnowledgeMapper.java`
2. 确认对应的数据库表主键定义为`AUTO_INCREMENT`

**需要检查的文件**：
```
src/main/java/com/sixth_group/finsight_backend/mapper/FormulaMapper.java
src/main/java/com/sixth_group/finsight_backend/mapper/ThresholdMapper.java
src/main/java/com/sixth_group/finsight_backend/mapper/KnowledgeMapper.java
```

## 7. 注意事项

1. 所有新建表必须使用`utf8mb4`字符集以支持中文
2. PDF报告表和预警记录表的insert语句需要使用`@Options`注解获取自增ID
3. 投资方案表的cash_flows字段使用JSON类型存储
4. 如果使用Flyway进行数据库迁移，确保版本号连续
